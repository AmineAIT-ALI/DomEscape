DOCUMENTATION TECHNIQUE

Référence technique

Endpoints API, formats de payload, intégration Domoticz, scripting dzVents et microservice LCD.

ENDPOINTS API

Trois endpoints. L'interface complète.

POST /api/handle_event.php Endpoint webhook

Appelé par dzVents à chaque événement capteur. Valide le token, normalise le payload et exécute le moteur de scénarios.

Requête
POST
POST /api/handle_event.php
token=domescape_secret_2025
idx=5
nvalue=1
svalue=
Réponse
JSON
// succès
{ "status": "ok",
  "code_evenement": "BUTTON_PRESS",
  "capteur": "Fibaro Button" }
// ou ignoré
{ "status": "ignored",
  "message": "Événement non reconnu..." }

Note : Token validé contre la constante WEBHOOK_TOKEN. Retourne 403 si token invalide.

GET /api/session_status.php Polling frontend — intervalle 1s

Retourne l'état complet de session pour le polling JS. Appelé toutes les secondes par player.php et gamemaster.php.

Réponse
Réponse JSON
{
  "status": "en_cours",
  "session_id": 12,
  "joueur": "Équipe Alpha",
  "scenario": "DomEscape Lab 01",
  "score": 350,
  "nb_erreurs": 2,
  "elapsed_seconds": 252,
  "etape": {
    "id": 3,
    "numero": 3,
    "titre": "Scan de mouvement",
    "description": "Traversez la zone de détection.",

Note : Retourne {"status":"no_session"} si aucune session active.

POST /api/start_game.php Initialisation de session

Crée une nouvelle session de jeu. Réinitialise d'abord toute session active existante. Crée l'enregistrement joueur s'il n'existe pas.

Requête
POST
POST /api/start_game.php
id_scenario=1
nom_joueur=Équipe Alpha
Réponse
JSON
{
  "status": "ok",
  "id_session": 12,
  "message": "Partie démarrée."
}
DOMOTICZ

Intégration dzVents.

Un seul script dzVents écoute tous les capteurs par nom et envoie des webhooks POST.

dzVents — domescape.lua
local SENSOR_NAMES = {
  'Fibaro Button', 'Door Sensor', 'Multisensor'
}
local WEBHOOK_URL = 'http://localhost/domescape/api/handle_event.php'
local TOKEN = 'domescape_secret_2025'
return {
  on = { devices = SENSOR_NAMES },
  execute = function(dz, device)
    dz.openURL({
      url = WEBHOOK_URL,
      method = 'POST',
      postData = {
        token = TOKEN,
        idx = device.idx,
        nvalue = device.state == 'On' and 1 or 0,
        svalue = device.rawData[1] or ''
      }
    })
  end
}
Notes d'intégration
· Les noms dans SENSOR_NAMES doivent correspondre exactement aux noms des périphériques Domoticz
· Les actionneurs (lampe, prise) NE doivent PAS figurer dans SENSOR_NAMES — évite les boucles d'événements
· Mapping nvalue : porte/mouvement utilisent 0/1 ; bouton utilise 1 (simple), 2 (double), 3 (triple), 10 (maintien)
· Installation : Domoticz → Paramètres → Événements → Nouveau script
MICROSERVICE LCD

Service Python Flask LCD.

LCD PiFace CAD sur bus SPI. Tourne comme processus Flask séparé sur le port 5000 pour éviter les problèmes de permissions SPI de www-data sur Apache.

Endpoint Méthode Description
/lcd?msg=TEXTE GET Afficher un message sur le LCD (16x2 caractères)
/lcd/clear GET Effacer l'écran LCD
/health GET Vérification de disponibilité du service
Note : En mode simulation (sans matériel PiFace), les messages sont seulement journalisés. Utile pour le développement hors Raspberry Pi.
lcd_service.py
# ActionManager appelle :
GET http://localhost:5000/lcd?msg=Syst%C3%A8me+en+ligne.
# Route Flask :
@app.route('/lcd')
def lcd():
  msg = request.args.get('msg', '')
  cad.lcd.write(msg[:32])
  return jsonify({
    'status': 'ok',
    'message': msg
  })
