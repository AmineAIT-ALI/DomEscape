// Project Vision

Domotics can do
more.

Home automation was designed to make buildings respond to people. DomEscape takes this premise further: what if a building could tell a story, set a challenge, and reward cleverness?

Why DomEscape Exists

The paradigm shift from control to experience.

Every Z-Wave sensor is, at its core, a detector of human intent. A door opens because someone decided to open it. A button is pressed because someone made a choice. A motion sensor fires because a person moved through a space.

Traditional home automation treats these events as operational signals: close the blinds, turn on the heater, log the timestamp. DomEscape treats them as narrative events — meaningful steps in an interactive story that unfolds in physical space.

The result is a platform that demonstrates what event-driven automation can become when the application domain shifts from building management to immersive human experience.

THESIS
"The same Z-Wave sensor that automates a blind can validate an enigma. The infrastructure is identical. The intention is not."
THE SHIFT
Smart-home dashboard → Interactive physical narrative
Hardcoded escape room → Database-configurable platform
Read-only sensor monitoring → Event-driven game logic

Future Directions

Where DomEscape could go.

The V1 platform proves the concept with a linear four-puzzle sequence. The architecture anticipates much richer experiences.

Near-term
Multi-room sessions
Route events by room — multiple teams, multiple simultaneous sessions, independent sensor pools.
Near-term
Branching scenarios
puzzle_transitions enables different paths depending on whether a player succeeded or failed — narrative consequences.
Medium-term
Scenario marketplace
Export/import game configurations as JSON packages. Share community-built scenarios across installations.
Long-term
Analytics engine
Aggregate session_events across all sessions to identify which puzzles are too hard, too easy or broken.
