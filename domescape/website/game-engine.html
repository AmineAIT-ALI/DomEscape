Why stateless?

PHP is stateless by design — each HTTP request spawns a fresh process. Rather than fighting this constraint with session files or shared memory, DomEscape embraces it.

Every call to GameEngine::process() starts by reading the complete game state from MariaDB. This makes every call idempotent, debuggable, and resilient to crashes.

What the engine holds in memory

// GameEngine::process() — per-request scope

$session = getActiveSession(); // DB read

$expected= getExpectedEvent($puzzle['id_puzzle']);

if (matchesExpected($event, $expected)) {
  onSuccess($session, $puzzle); // DB write
} else {
  onFailure($session, $puzzle); // DB write
}

matchesExpected() — V1 logic

A match is valid when both the sensor and the event type match what is defined in puzzle_expected_events for the current puzzle.

// Matching algorithm
if ($expected['id_sensor'] !== $event['sensor']['id_sensor'])
  return false; // wrong sensor

if ($expected['id_event_type'] !== $event['event_type']['id_event_type'])
  return false; // wrong event type

return true; // ✓ puzzle validated

Action System

Four trigger moments. Unlimited actions.

on_enter
Puzzle Start
Triggered when the session transitions to this puzzle. Typically shows a clue on the LCD display.
on_success
Puzzle Solved
Triggered on correct event. Can activate a plug and display a congratulatory message.
on_failure
Wrong Action
Triggered on unmatched event. Increments mistake_count and can display an error message on LCD.
on_hint
Hint Requested
Triggered by Game Master intervention. Displays a pre-configured hint without advancing the game state.

Roadmap

The engine is designed to grow.

V1 implements linear sequential puzzles — one event, one outcome. The architecture anticipates more complex interactions without breaking the existing engine contract.

V2 — Timed Puzzles
Time-limited enigmas
Puzzles with a countdown — wrong answer or timeout triggers an alternate path or score penalty.
V2 — Branching
Non-linear scenarios
puzzle_transitions table enables success/failure paths to diverge, creating unique narrative branches.
V3 — Compound Events
Multi-sensor puzzles
Require simultaneous conditions across multiple sensors — e.g. door open AND motion detected within 5 seconds.
