Scenario Builder

A platform, not
a single game.

DomEscape decouples the game engine from game content. Scenarios live in the database. New escape games are created through configuration — never through code changes.

DB-Driven No Redeployment Sensor-Agnostic
Platform Thinking

One engine. Infinite scenarios.

The game engine interprets configuration — it doesn't encode game rules. Swap the database rows, and you have a completely different escape game.

∞
Infinite Scenarios
Any sequence of Z-Wave sensor events can be a puzzle sequence. The engine doesn't know or care what the story is.
0
Zero Code Changes
Deploying a new game scenario requires only database operations. The PHP engine doesn't change. The dzVents script doesn't change.
5
Device Types
Door sensor, motion sensor, button (multi-press), wall plug, LCD — every physical device can play a role as input or output in any scenario.
