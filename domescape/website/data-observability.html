MODÈLE DE DONNÉES

Deux domaines. Un schéma.

La base de données est divisée entre la configuration de conception (ce qui doit se passer) et l'état runtime (ce qui s'est réellement passé).

Conception (Statique)
scenario etape capteur actionneur evenement_type action_type etape_attend etape_declenche

Configuré avant le démarrage du jeu. Définit les règles.

Runtime (Dynamique)
joueur session evenement_session action_executee

Écrit pendant le jeu. Enregistre ce qui s'est passé.

ANALYTIQUE

Les données répondent aux questions.

Quelle étape est la plus difficile ?

Compter les événements d'erreur par étape sur toutes les sessions. Taux d'erreur élevé = problème de conception ou instruction peu claire.

Durée moyenne de session

SELECT AVG(duree_secondes) FROM session WHERE statut = 'gagnee'. Comparaison avec le temps cible du scénario.

Taux de réussite par scénario

Ratio de sessions gagnee sur abandonnee + perdue. Mesure l'équilibre global et l'accessibilité du scénario.

analytics.sql
-- Quelle étape provoque le plus d'erreurs ?
SELECT e.titre_etape, COUNT(*) AS erreurs
FROM evenement_session es
JOIN etape e ON es.id_etape = e.id_etape
WHERE es.valide = 0
GROUP BY e.id_etape
ORDER BY erreurs DESC;
