Data Model

The database is
the game engine's memory.

State, puzzles, sessions, sensors and logs — everything persists in MariaDB. The stateless PHP backend is nothing without a well-designed relational schema.

9 Tables MariaDB / MySQL PDO / PHP Full FK Integrity
Schema Overview

Two planes. One schema.

The schema cleanly separates game configuration (design-time) from game execution (runtime). Modifying one doesn't affect the other.

Design-time — Game Configuration
sensors Z-Wave device registry with domoticz_idx
sensor_events_catalog Normalized event type dictionary
games Escape game metadata
puzzles Ordered enigmas within a game
puzzle_expected_events The action that validates a puzzle
puzzle_actions Physical feedback on trigger events
Runtime — Execution & History
players Teams and individual players
game_sessions Active and past sessions with full state
session_events Complete event journal — raw + normalized
Entity Relationships
games 1─N puzzles
puzzles 1─N puzzle_expected_events N─1 sensors
puzzles 1─N puzzle_actions N─0 sensors
sensor_events_catalog 1─N puzzle_expected_events

players 1─N game_sessions N─1 games
game_sessions 0─1 puzzles (current_puzzle_id)
game_sessions 1─N session_events
session_events N─1 sensors
session_events N─1 sensor_events_catalog
