CENTRE DE CONTRÔLE

Trois interfaces.
Un runtime.

Joueur, Supervision et Administration disposent chacun d'une interface dédiée. Les trois lisent le même état de session dans MariaDB.

INTERFACE JOUEUR

Le feedback passe par l'environnement.

Le joueur n'a pas d'écran. Le feedback physique est l'interface : la prise s'active, le LCD affiche des messages. La vue web est destinée aux spectateurs ou à un projecteur.

player.php — polling toutes les 1s
Équipe : Équipe Alpha 04:12 Score : 350
●●●○ Étape 3/4 ✗ 2 erreurs
SCAN DE MOUVEMENT
Traversez la zone de détection pour compléter le scan.
Le capteur est au centre de la pièce.

JS interroge /api/session_status.php toutes les 1000ms. Pas de WebSocket nécessaire. La réponse JSON contient l'état de session, les données de l'étape courante, le score, le nombre d'erreurs et le temps écoulé.

SUPERVISION

Suivi de session en direct.

La supervision voit tout en temps réel : étape courante, score, erreurs, temps écoulé et journal d'événements. Les contrôles permettent la réinitialisation et l'envoi d'indices.

ÉDITEUR DE SCÉNARIO

Les étapes se configurent, ne se codent pas.

Chaque étape définit quel événement capteur la valide, quelles actions se déclenchent selon le résultat, et quels messages afficher.
