POURQUOI DOMESCAPE

Deux secteurs. Les mêmes capteurs.
Zéro programmabilité.

La domotique réagit automatiquement. Les escape rooms sont programmées manuellement. Aucun système ne permet de définir la logique sous forme de scénarios configurables.

Maisons connectées limitées

La domotique réagit aux événements avec des règles fixes et codées en dur. Aucune logique narrative, aucune progression, aucun état.

Escape rooms figées

Les escape rooms traditionnelles nécessitent un recâblage physique pour chaque scénario. Changer une énigme implique de modifier l'installation.

Pas de plateforme d'interaction

Aucun système ne traite les événements capteurs comme des primitives d'interaction programmables capables de piloter des narrations configurables.

LA PLATEFORME

Événements en entrée. Réactions en sortie.
Logique au milieu.

Les capteurs détectent les actions physiques. Le moteur les interprète en fonction d'un scénario chargé depuis la base de données. Si valide, l'étape avance et les actionneurs réagissent.

Le scénario est une donnée — interchangeable sans toucher au code. Un nouveau jeu, c'est un nouvel ensemble d'enregistrements en base, pas un nouveau déploiement.

Chaque événement est enregistré. Chaque transition est persistée. Chaque action est traçable. Le moteur est constant ; l'expérience est variable.

CONCEPTS CLÉS

Six idées qui définissent la plateforme.

Architecture événementielle

Chaque interaction commence par un événement physique. Les capteurs émettent, le moteur interprète, les actionneurs répondent.

Logique pilotée par la base de données

Scénarios, étapes, capteurs et événements attendus sont tous stockés dans MariaDB. Aucun changement de code pour créer un nouveau jeu.

Boucle de feedback physique

Les actions correctes déclenchent de vraies réactions physiques : messages LCD, prise activée.

Supervision en temps réel

Une boucle de polling à 1 seconde maintient l'interface web synchronisée avec l'état de session en permanence.

Séparation des responsabilités

EventManager, GameEngine et ActionManager sont des modules entièrement découplés à responsabilité unique.

Scénarios configurables

Chaque étape, déclencheur, action et séquence est configurable via la base de données. Un moteur, une infinité de scénarios.

INGÉNIERIE

Construit sur des principes solides.

Backend stateless
PHP + MariaDB

PHP lit l'état depuis MariaDB à chaque requête. Aucune session en mémoire. Résistant aux pannes et horizontalement scalable.

Middleware IoT
Domoticz + dzVents

Domoticz abstrait toute la complexité Z-Wave. dzVents relie la couche IoT à la couche applicative via webhooks.

Observabilité totale
100% tracé

Chaque événement, validation, action et transition de session est enregistré dans la base pour le debug et la rejouabilité.

Modèle plateforme
∞ Scénarios

L'escape game de démonstration est un scénario parmi d'autres. Le moteur peut exécuter n'importe quelle narration pilotée par capteurs sans modification.
