PLATEFORME

Un moteur.
Une infinité de scénarios.

DomEscape n'est pas un jeu. C'est un moteur d'interaction configurable. Le scénario est une donnée. Le moteur est constant. Seule la configuration change.

BOUCLE PRINCIPALE

Événement physique → Validation → Réaction

Étape 1
Espace physique

Un capteur détecte une action réelle. Porte ouverte. Bouton pressé (simple ou double). Mouvement détecté.

ÉVÉNEMENT CAPTEUR
Étape 2
Moteur de scénarios

Le moteur charge la session courante depuis MariaDB. Il compare l'événement entrant avec la paire capteur/type_événement attendue. Correspondance ou non.

DÉCISION MOTEUR
Étape 3
Réaction physique

En cas de succès : score mis à jour, étape suivante chargée, actionneurs déclenchés (LCD, prise). En cas d'échec : compteur d'erreurs incrémenté.

SORTIE ACTIONNEUR
ÉTAT DE SESSION

La session comme machine à états

Une session évolue selon des états bien définis. Chaque transition est déclenchée par un événement physique et persistée dans MariaDB.

stateless design
// PHP est stateless. Chaque requête lit l'état complet de session depuis MariaDB.
// Le moteur ne conserve aucun état en mémoire.
$session = Session::loadActive(); // lecture BDD à chaque fois
$etape = Etape::loadCurrent($session);
$attend = EtapeAttend::loadFor($etape);
// Aucun état en mémoire. Résistant aux pannes. Redémarrable à tout moment.
CONFIGURATION

Les scénarios sont des données, pas du code.

Chaque aspect d'un scénario est stocké dans la base de données. Créer un nouveau jeu ne nécessite ni déploiement ni modification du code — seulement des enregistrements en base.

Entité Ce qu'elle définit
scenario Nom, thème, description
etape Ordre de l'étape, titre, points, est_finale
etape_attend Quel capteur + événement valide chaque étape
etape_declenche Quelles actions se déclenchent en on_enter / on_success / on_failure
capteur Périphérique Z-Wave identifié par son domoticz_idx
actionneur Périphérique de sortie (prise murale, LCD)
RÔLES

Trois rôles. Un système.

Joueur

Interagit avec l'espace physique. N'a pas accès à l'interface. Reçoit le feedback via les actionneurs : prise murale, messages LCD. Immersion totale.

Supervision

Surveille la session en temps réel via la console de supervision. Peut réinitialiser la session, envoyer des indices et observer tous les événements en direct.

Administration

Configure les scénarios, mappe les capteurs, gère la base de données. Accès complet au tableau de bord d'administration et à l'éditeur de scénarios.
