VISION

Le même capteur qui
automatise un volet peut
valider une énigme.

DomEscape démontre que l'infrastructure domotique peut servir des cas d'usage radicalement différents quand la couche d'abstraction change.

L'ARGUMENT

Ce n'est pas un escape game.
C'est un argument de plateforme.

Les plateformes domotiques optimisent pour le confort et l'automatisation. Chaque événement capteur déclenche un script prédéfini : mouvement détecté → allumer la lumière. La logique est simple car l'objectif est simple.

DomEscape pose une question différente : et si les événements capteurs n'étaient pas des déclencheurs pour des scripts d'automatisation, mais des entrées d'un moteur de scénarios programmable ? Et si la signification d'un événement capteur dépendait de l'état courant d'une session — et pas seulement de l'événement lui-même ?

Le scénario d'escape game est la démonstration la plus claire possible : une porte qui s'ouvre ne signifie rien prise isolément. Mais si c'est l'action attendue à l'étape 2 de la session #12, elle fait avancer le jeu, met à jour le score et déclenche une cascade de réactions physiques. Le même matériel. Une abstraction entièrement différente.

Cette abstraction — les événements comme entrées de scénario, non comme déclencheurs d'automatisation — est ce que DomEscape est conçu pour prouver.

CHANGEMENTS DE PARADIGME

Avant et après le changement d'abstraction.

avant → après
Maison connectée → Narration interactive

Avant : mouvement détecté → allumer la lumière (automatique).

Après : mouvement détecté → valider l'étape 3 (contextuel, avec état, piloté par scénario).

avant → après
Câblage fixe → Configurable

Avant : câblage physique pour chaque énigme (une installation = un jeu).

Après : des enregistrements en base définissent toutes les règles (une installation = une infinité de jeux).

avant → après
Lecture seule → Événementiel

Avant : les capteurs rapportent l'état.

Après : les capteurs émettent des événements qui pilotent la progression narrative. L'espace devient programmable.

APPLICATIONS

Le même moteur. Des contextes différents.

Tout espace équipé de capteurs et d'actionneurs peut devenir un environnement interactif et programmable.

Installations muséales

Les capteurs activent des contenus audio, d'éclairage ou d'affichage contextuels lorsque les visiteurs se déplacent dans les espaces. Chaque interaction est enregistrée et séquencée.

Simulations de formation

Tâches physiques validées par capteurs. Les apprenants reçoivent un feedback en temps réel via les actionneurs. Progression suivie par session.

Théâtre interactif

Acteurs et public interagissent avec une scène équipée de capteurs. Les transitions de scènes sont événementielles, pas temporelles.

Laboratoires pédagogiques

Exercices de TP validés par capteurs. Les étudiants prouvent qu'ils ont réalisé la bonne séquence. Évaluation automatisée.

Intégration industrielle

Les nouveaux employés réalisent des séquences de mise en service physiques validées par événements capteurs. Certification sans papier.

Divertissement immersif

Salles VR ou AR où les événements capteurs physiques se synchronisent avec des superpositions numériques.

ROADMAP

La V1 est le socle.

MAINTENANT — V1
Scénarios linéaires

Session active unique. Étapes séquentielles. Un événement attendu par étape. Observabilité totale. Architecture plateforme complète.

PROCHAIN — V2
Embranchements + Minuterie

Scénarios multi-chemins. Étapes chronométrées. Événements composites (plusieurs capteurs simultanément). Système d'indices.

FUTUR — V3
Multi-salle + Analytique

Plusieurs salles en réseau. Tableau de bord analytique. Visualisation du replay de session. Marketplace de modèles de scénarios.

HORIZON — V4
Plateforme API

API REST pour l'authoring externe de scénarios. Application mobile de supervision. Intégration de capteurs tiers au-delà du Z-Wave.

En résumé

DomEscape transforme un espace physique en système interactif programmable.

Ni jeu, ni maison connectée. La démonstration que l'infrastructure est réutilisable quand l'abstraction est juste. Les mêmes capteurs, le même middleware, le même Raspberry Pi — mais un système entièrement différent parce que la couche logique a changé.

C'est la pensée d'ingénierie appliquée à l'espace physique.
